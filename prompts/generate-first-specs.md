Using a bounded brownfield discovery result as evidence, create the smallest
set of project-specific first-pass product specs needed before implementation.

Read `AGENTS.md`, `docs/spec-first-workflow.md`, `docs/specs/README.md`, and
`docs/specs/index.md` first. Use `docs/specs/templates/feature-spec.md` and
make every spec short, product-level, and explicit about scope, non-goals,
user-visible behavior, invariants, edge cases, route/state/data implications,
verification mapping, and unknowns requiring confirmation.

Do not change product implementation in this pass. Code, tests, and runtime
evidence may reveal behavior but do not decide intent where product materials
conflict.
